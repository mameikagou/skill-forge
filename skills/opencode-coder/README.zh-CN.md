# skill-forge

Claude Code 技能工坊 — 让 Claude 自身变得更强的工具集。

## 技能列表

| 技能 | 状态 | 描述 |
|------|------|------|
| [opencode-coder]() | ✅ 可用 | 把代码生成委派给便宜模型（MiniMax、Codex），节省 95% 成本 |
| [write-skill](../write-skill) | 📦 占位 | 教 Claude 写出符合 Anthropic 规范的 skill |
| [skill-auto-installer](../skill-auto-installer) | 📦 占位 | 将任意 skill 打包为一键安装格式 |
| [code-graph](../code-graph) | 📦 占位 | 构建本地 LSP 代码图谱，高效阅读大型代码库 |

## 开始使用

```bash
# 安装依赖（链接 workspace）
bun install

# 运行全部测试
bun test

# 运行单个 skill 的测试
cd skills/opencode-coder && bun test
```

## 目录结构

```
skill-forge/
├── package.json              # Bun workspace 根配置
├── packages/
│   └── shared/               # 共享工具库 (@skill-forge/shared)
└── skills/
    ├── opencode-coder/       # 成本套利：Opus 思考，便宜模型干活
    ├── write-skill/          # 元技能：教 Claude 正确写 skill
    ├── skill-auto-installer/ # 把 skill 打包成可分发格式
    └── code-graph/           # 基于 LSP 的代码图谱
```

## 何时使用？

  第一步：大白话拆解 /opencode-coder 的省钱逻辑

  核心思路就像一个公司老板和外包团队的关系：

  Opus（老板）：看需求、做决策、审查结果 → 贵，但只花少量 token
  MiniMax（外包）：写大量代码 → 便宜，干苦力活

  省钱的关键在于：代码生成是 output token，这是最贵的部分。

  ┌────────────────────┬───────────┬──────────────┬───────────────┐
  │     Token 类型     │ Opus 价格 │ MiniMax 价格 │     差距      │
  ├────────────────────┼───────────┼──────────────┼───────────────┤
  │ Output（生成代码） │ $15/M     │ $2.4/M       │ Opus 贵 6.25x │
  ├────────────────────┼───────────┼──────────────┼───────────────┤
  │ Input（读代码）    │ $3.75/M   │ —            │ 这部分不能省  │
  └────────────────────┴───────────┴──────────────┴───────────────┘

  第二步：具体成本对比

  场景：改写 3 个文件，~8000 tokens 的代码输出

  ┌────────────────┬────────────────────────┬────────────────┬────────┬──────┐
  │      方式      │      Opus Output       │ MiniMax Output │ 总成本 │ 节省 │
  ├────────────────┼────────────────────────┼────────────────┼────────┼──────┤
  │ 纯 Opus        │ 8,000 tokens           │ —              │ $0.120 │ —    │
  ├────────────────┼────────────────────────┼────────────────┼────────┼──────┤
  │ opencode-coder │ ~50 tokens（派发指令） │ 8,000 tokens   │ $0.020 │ 83%  │
  └────────────────┴────────────────────────┴────────────────┴────────┴──────┘

  场景：一天 50 次文件修改

  ┌────────────────────────────┬──────────────────────────────────┬───────┐
  │            方式            │        Opus Output tokens        │ 成本  │
  ├────────────────────────────┼──────────────────────────────────┼───────┤
  │ 纯 Opus                    │ 400,000                          │ $6.00 │
  ├────────────────────────────┼──────────────────────────────────┼───────┤
  │ opencode-coder + Opus 审查 │ 7,500 (Opus) + 400,000 (MiniMax) │ $1.07 │
  └────────────────────────────┴──────────────────────────────────┴───────┘

  日均省 ~$5，月省 ~$100。

  第三步：什么时候用 /opencode-coder？

  你要改代码？
  ├─ 改 <10 行，1 个文件 → Opus 直接 Edit（不值得派发）
  ├─ 改 10-200 行，1 个文件 → /opencode-coder ✅
  ├─ 改 2-3 个文件联动 → /opencode-coder ✅（MiniMax 擅长）
  ├─ 改 4+ 个文件 → Opus 自己来（超出外部模型上下文）
  └─ 需要全局架构理解 → Opus 自己来（外部模型看不到其他文件）

  模型选择

  /opencode-coder minimax file1,file2 instruction  ← 快+便宜，日常首选
  /opencode-coder codex file1 instruction           ← 质量优先，复杂逻辑

  ┌───────────────┬───────────────┬───────────┐
  │     场景      │  选 MiniMax   │ 选 Codex  │
  ├───────────────┼───────────────┼───────────┤
  │ 加注释/文档   │ ✅ 中文好，快 │           │
  ├───────────────┼───────────────┼───────────┤
  │ 简单 bug 修复 │ ✅ 速度快     │           │
  ├───────────────┼───────────────┼───────────┤
  │ 模板代码/CRUD │ ✅ 100 tps    │           │
  ├───────────────┼───────────────┼───────────┤
  │ 2-3 文件联动  │ ✅ 成本低     │           │
  ├───────────────┼───────────────┼───────────┤
  │ 复杂算法重写  │               │ ✅ 推理强 │
  ├───────────────┼───────────────┼───────────┤
  │ 类型注解推断  │               │ ✅ 准确   │
  ├───────────────┼───────────────┼───────────┤
  │ 核心架构代码  │               │ ✅ 质量高 │
  └───────────────┴───────────────┴───────────┘

  第四步：完整的省钱组合拳

  把之前的调研和 opencode-coder 结合起来：

  省钱层级（从大到小）：

  1. 模型选择        → Sonnet 替代 Opus 做日常任务（省 5x）
  2. opencode-coder  → 代码生成交给 MiniMax（省 output 6.25x）
  3. Subagent 隔离   → 探索/搜索交给 Haiku（省 50-70% 探索开销）
  4. /compact        → 及时压缩（省 30-50% 上下文开销）
  5. 精简 CLAUDE.md  → 减少基线 token
  6. 禁用闲置 MCP   → 每个省 3-5%

  组合使用月省估算：$100-200 → $30-60

  你的最优工作流

  开始任务（用 Sonnet 或 Opus）
    ↓
  读项目、理解需求（用 Explore subagent / Haiku）
    ↓
  小改动 (<10行) → Opus/Sonnet 直接 Edit
  大改动 (10+行, 1-3文件) → /opencode-coder minimax
    ↓
  Opus 审查 diff → 确认质量
    ↓
  /compact → 压缩上下文
    ↓
  下一个任务...
    ↓
  完全不同的任务 → /clear

  踩坑预警

  1. opencode-coder 不是万能的 —— 它看不到你没传给它的文件，所以需要全局架构理解的任务，Opus 自己来更好
  2. MiniMax 偶尔格式不规范 —— runner.ts 有自动清理，但多文件时如果 XML 解析失败会报错（单文件有
  fallback）
  3. 不要对 .env / credentials 用 —— 内容会发给外部 API
