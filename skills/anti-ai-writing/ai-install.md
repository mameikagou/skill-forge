# 降低 AI 率与查重率 — 中文学术论文改写指南

## 技能目标

你的任务是接收用户提供的中文文本，将其改写为**低 AI 疑似度、低查重率**的学术文本。

**核心原理**：AI 检测工具（知网 AIGC 等）通过两个底层指标判定文本：
- **困惑度（Perplexity）**：AI 文本用词高度可预测（~47），人类文本用词更"意外"（~165）
- **突发性（Burstiness）**：AI 文本句长均匀、节奏平坦；人类文本长短句交替、节奏跳跃

你的改写必须打破 AI 生成时完美的概率分布模型，同时**严格保持学术严谨性**。

---

## 策略一：打碎八股文风（提升突发性 Burstiness）

### 1.1 消灭 AI 套话开头

以下开头模式必须彻底消灭，直接切入主题：

| 禁用套话 | 替代方案 |
|---------|---------|
| "随着…的不断发展" | 直接陈述现状或问题 |
| "在当今…背景下" | 省略，从具体事实起笔 |
| "近年来…引起了广泛关注" | 给出具体年份和具体事件 |
| "本文旨在探讨…" | 直接进入论述 |

### 1.2 消灭 AI 高频过渡词

绝对禁用以下过渡词，用更自然的衔接替代：

| 禁用 | 替代方案 |
|-----|---------|
| 首先、其次、最后 | 删除序号词，用逻辑关系自然衔接 |
| 此外、另外、与此同时 | "还有一点值得留意"、"同样不容忽视的是"、直接另起一段 |
| 综上所述、总而言之 | "由此来看"、"回过头看这几组数据"、直接给出结论 |
| 值得注意的是 | 删掉，直接说 |
| 不难发现 | "数据表明"、"从结果看" |

### 1.3 句式长短交替（核心）

这是降低 AI 率最有效的单一策略。严格执行：

- **短句**（8-15 字）与**长句**（30-50 字）必须交替出现
- 禁止连续 3 句以上句长相近（差值 < 5 字）
- 每段至少包含 1 个极短句（< 10 字）作为节奏断点
- 示例："这一假设并不成立。实验数据显示，当温度超过临界阈值后，催化剂的活性不仅没有按照理论预期呈线性增长，反而出现了明显的衰减拐点。原因何在？"

### 1.4 引入非常规句式

在保持学术语气的前提下，适当加入：

- **设问句**："这是否意味着传统模型已经失效？"
- **反问句**："如果连基准条件都无法满足，后续实验的可靠性又从何谈起？"
- **倒装强调**："恰恰是这一被忽视的变量，决定了最终结果的走向。"
- **插入语**："该方法——尽管在小规模数据集上表现优异——在实际部署中暴露了严重的泛化问题。"
- **破折号补充**："精度提升了 12%——这个数字本身并不惊人，但背后的机制值得深究。"

### 2.1 AI 高频词黑名单与替换表

| AI 高频词 | 替换为 |
|----------|-------|

| 范式 | 模式、路径、做法 |
| 维度 | 角度、层面、方面 |
| 系统性 | 整体的、全面的、一揽子 |
| 深层次 | 根本的、核心的 |
| 多维度 / 全方位 | 多个角度、各方面 |

| 具有重要意义 | 意义不小、值得重视 |

| 新增高频词 | 替换为 |

| 具有…特征 | 呈现出…特点 |
| 从…角度出发 | 立足于… / 以…为切入点 |

### 2.3 注入自然语感连接词

在段落衔接处适当加入带有人类思考痕迹的连接词，但**不能过于口语化以致损害学术性**：

**适合学术论文的自然连接**：
- "实际上"、"事实上"
- "有意思的是"、"耐人寻味的是"
- "换一个角度看"、"如果换个思路"
- "不得不承认"、"坦率地说"
- "笔者注意到"、"据笔者观察"

**禁止使用**（过于口语）：
- "说白了"、"讲真"、"你想啊"

### 2.4 核心术语保护

**绝对不能修改的内容**：
- 学科专业术语（如"卷积神经网络"、"边际效用递减"）
- 方法论名称（如"扎根理论"、"德尔菲法"）
- 固定搭配的学术表达（如"显著性水平"、"置信区间"）
- 人名、机构名、数据集名称
- 直接引用的原文

## 策略三：降低查重率专项（含语言学改写技巧）

### 3.1 句式重构（核心手段）

知网查重以**连续 13 字以上重复**为标红阈值。以下变换可打断连续匹配：

| 变换类型 | 示例 |
|---------|------|
| 主被动转换 | "该方法提高了检测精度" → "检测精度因该方法的引入而得到改善" |
| 因果倒置 | "由于样本量不足，实验结果存在偏差" → "实验结果出现偏差，根源在于样本量的不足" |
| 长句拆分 | 将 40 字以上的复合句拆为 2-3 个短句 |
| 短句合并 | 将 2-3 个相关短句合并为一个带从句的长句 |
| 定语移位 | "基于深度学习的目标检测方法" → "目标检测方法，尤其是以深度学习为基础的那一类" |
| 列举 ↔ 叙述 | 将"A、B、C 三个因素"改为叙述式分别展开 |

### 3.2 语义等价重写

不是简单的同义词替换，而是**用自己的理解重新表达同一个意思**：

- 将抽象表述**具体化**："效果显著" → "准确率从 78.3% 跃升至 91.7%"
- 将具体表述**抽象化**：罗列的具体数据 → 归纳为趋势性描述
- 将正面论述**反面化**："该方法优于传统方法" → "传统方法在此场景下的局限性，恰好被该方法所弥补"

### 3.3 结构重组

- 调整论述段落的内部顺序（不改逻辑，改呈现方式）
- 把"背景→方法→结论"偶尔改为"结论前置→方法回溯→背景补充"
- 将文献综述从"逐篇罗列"改为"按观点聚类对比"
- 在引用他人观点后，附带一句主观评价："这一发现虽然具有参考价值，但其实验条件与本研究存在差异"

### 3.4 语言学层面改写技巧（中文特有）

#### 3.4.1 语体升降与语体混合

- **语体降级**：将纯书面语表述适度混入口语化元素，打破 AI 文本的"标准书面语"特征
  - "该系统具备较强的鲁棒性" → "这套系统在实际运行中表现得相当皮实"
  - "上述结果表明" → "从上面的结果来看"
- **语体升格**：将口语化表述适度文言化，增加表达陌生感
  - "非常有用" → "裨益甚大"、"颇具参考价值"
  - "很重要" → "至为关键"、"不可或缺"

#### 3.4.2 语序重构（话题-述语结构）

汉语是"话题突出"语言，不同于英语的"主语突出"。利用这一特点进行改写：

- **话题前置**："深度学习方法在图像识别领域取得了突破" → "图像识别领域，深度学习方法取得了突破"
- **状语位移**：将时间/地点状语前置或后置，打破固定语序
  - "2023 年，该团队提出了新模型" → "该团队于 2023 年提出了新模型"
- **补语前置强调**："跑得很快" → "很快——这是它的速度"

#### 3.4.3 成分隐现与增添

- **省略主语**：在上下文明确时省略主语，增加阅读跳跃感
  - "本文分析了三种方法。我们发现，方法 A 的精度最高" → "本文分析了三种方法。精度最高的，是方法 A"
- **增添插入语**：在句中插入第一人称评价或思考痕迹
  - "实验结果表明" → "实验结果——至少从当前的数据来看——表明"
- **补语拆分**：将结果补语独立成句
  - "计算结果证明该假设成立" → "计算结果出来了。该假设，确实成立"

#### 3.4.4 修辞格变换

- **比喻 ↔ 直述**：将隐喻改为直白陈述，或将直白陈述改为隐喻
  - "数据呈指数增长" → "数据像滚雪球一样越滚越大"
- **反讽/反问**：适度引入反问句，增加语气波动
  - "该方法效果更好" → "该方法效果更好，这一点难道还有疑问？"
- **排比打散**：AI 喜欢排比，打散它
  - "提高了效率、降低了成本、增强了稳定性" → "效率提上去了，成本也压下来了，稳定性跟着水涨船高"

#### 3.4.5 文言/方言元素注入

适度引入古语词或方言词汇，打破 AI 文本的"普通话标准化"特征：

- 古语词替换："非常重要" → "至关重要"、"举足轻重"；"开始" → "伊始"、"肇始"
- 方言副词点缀："很好" → "蛮好"、"挺好"；"不知道" → "不晓得"、"不太清楚"
- **注意**：仅用于非核心论述的衔接处，专业段落保持规范书面语

### 3.5 结构对称性打破

多章节结构镜像对称是 AI 文本的强指纹信号。当论文中多个并行小节采用完全相同的内部结构（如"定义→原理→架构→场景→优缺点"），检测器会将其标记为 AI 生成。

**对称结构风险场景**：
- 技术对比章节：A、B、C 三个技术各 750 字，结构完全一致
- 实验对比章节：多组实验的描述段落结构相同
- 文献综述：每篇文献都是"作者+年份+方法+结论"的固定模板

**打破方法**：

| 方法 | 示例 |
|------|------|
| **错位排列** | 技术A：定义→原理→架构→场景→优缺点；技术B：场景→定义→优缺点→架构→原理 |
| **合并/拆分** | 将两个技术的"优缺点"合并为一段对比论述，而非各自独立成段 |
| **角度转换** | 技术A从"架构"角度切入，技术B从"应用场景痛点"切入 |
| **详略交替** | 技术A详细展开原理，技术B重点描述实际部署中的坑 |
| **叙事穿插** | 在技术描述中插入选型时的思考过程或踩坑经历 |

**操作原则**：
- 并行小节之间，内部结构的相似度不得超过 60%
- 每增加一个小节，至少改变一个核心论述顺序
- 允许某些小节"不完整"——不必每个都覆盖全部维度

### 3.6 特殊内容的降 AI 率策略

论文中的表格、公式、代码、参考文献等特殊内容同样需要处理：

| 内容类型 | AI 风险点 | 处理策略 |
|---------|----------|---------|
| **表格** | 表头描述文字（"如下表所示"等套话） | 替换描述文字，加入具体数据解读；表格上方文字重写 |
| **公式** | 公式推导过程的描述性文字 | 保护公式本身，改写推导说明 |
| **代码** | 代码注释和说明文字 | 保护代码块（建议先删除再处理），改写注释为自然语言描述 |
| **参考文献** | 引用格式过于标准 | 检查引用格式是否完全一致，适度使用直接引用 vs 间接引用 |
| **图表编号** | 知网 3.0 新增格式敏感检测 | 确保图表编号格式不是完美标准化；坐标轴单位不可缺失 |
| **脚注/尾注** | AI 不擅长生成脚注 | 添加适当脚注，增加人类写作痕迹 |
| **测试用例** | 多项用例格式完全统一（编号/模块/步骤/预期/结果） | 在描述字段加入个人处理经验、异常记录、踩坑细节；打破格式统一 |

**处理原则**：
1. **先保护再处理**：将代码块、数学公式替换为占位符（如 `[CODE_BLOCK]`），改写后再恢复
2. **表格描述重写**：表格上方的描述文字是 AI 率高发区，必须逐句改写
3. **公式参数标注**：知网 3.0 会检测公式中未标注的参数来源，确保每个参数都有说明

## 策略四：注入学术个性（人类写作标记）

AI 文本最大的破绽是**缺少作者的主体性**。以下手段可有效注入"人味"：

### 4.1 第一人称学术表达

- "笔者认为"、"本研究发现"、"据笔者了解"、"在笔者看来"

**使用策略**：
- 每 500-800 字出现 1 次第一人称标记
- 优先用于：研究局限性说明、方法选择理由、对研究结果的解读
- 避免用于：文献综述（除非是批判性评价）、公认事实的陈述

### 4.2 让步与反驳

- "然而也有学者持不同看法"、"尽管如此，该方法仍存在局限"、"这一结论并非没有争议"

### 4.3 研究局限性坦承

- "受限于数据规模，本研究的结论仍需进一步验证"（AI 很少主动承认局限）

### 4.4 具体化模糊引用

将"许多学者认为"改为"张三（2023）和李四（2024）均指出"
将"相关研究表明"改为具体的引用+页码

### 4.5 补充微观细节

为核心论点补充具体的实验困难、数据侧面或研究过程中的观察：
- "在第三轮实验中，由于设备校准偏差，我们一度得到了相反的结果"

### 4.6 更多注入学术个性的具体方法

| 方法 | 示例 |
|------|------|
| **研究动机个人化** | "本研究的出发点源于笔者在某次田野调查中观察到的一个现象" |
| **方法选择说明** | "之所以选择该方法而非传统的 X 方法，是出于以下考虑" |
| **过程中的意外** | "最初我们假设 A 与 B 正相关，但预实验结果恰好相反" |
| **对数据的直觉判断** | "尽管统计结果显示显著，但从实际意义看，这一差异恐怕难以产生实质性影响" |
| **学术史叙事化** | "这一领域的研究可以追溯到 20 世纪 80 年代，当时 Smith 首次提出了..." |

### 4.7 保留已有个人化痕迹

AI 改写最常见的副作用是**过度"学术化"**，把原本有人味的表达抹成四平八稳的八股文。以下痕迹必须保留：

**应保留的表达类型**：
- **项目黑话**：系统内部命名（如 `FABRIC_MOCK`、`最小上链原则`）——这是人类作者的专属记忆
- **具体数字**：精确到小数点的数据（"0.02 秒延迟"、"174 次引用"）——AI 常生成整数或约数
- **个人观察**："笔者注意到"、"据笔者观察"、"在调试过程中发现"

**禁止行为**：
- ❌ 把括号注释删掉，改为正式表述
- ❌ 把具体数字四舍五入为"约 170 次"
- ❌ 把项目内部术语替换为标准学术用语

**原则**：人味 > 学术腔。检测器对"太完美的学术表达"反而敏感。

## 策略五：检测工具机制与定向对抗

### 5.1 知网 AIGC 检测（2024-2025 最新）

#### 知网 AIGC 检测 3.0 版本（2025 年升级）

**技术原理**：基于百万级学术语料库，语义特征分析与模式识别。2025 年升级"语义指纹"技术，识别概率分布特征。

**三大核心升级**：

| 升级项 | 旧算法 | 新算法 |
|--------|--------|--------|
| **双链路检测** | 仅分析语言模式 | 新增"语义逻辑"链路，识别线性 A→B→C 结构 |
| **语义指纹识别** | "看词" | "看逻辑"——计算句长分布、词频波动、困惑度等统计学指纹 |
| **格式敏感检测** | 无 | 标准化标题层级触发预警；图表坐标轴缺失检测；公式参数来源检测 |

**新增能力**：
- 每两周更新模型
- "风格迁移检测"：可识别经 AI 改写工具处理的内容
- 单纯同义词替换仅能降低 10-15% AI 率，无法突破 40% 阈值

**新增检测维度（2025-2026）**：
- **词汇密度（Lexical Density）**：AI 文本词汇密度偏低（内容词/总词数比例 ~0.35），人类文本 ~0.45–0.55。增加实词密度、减少虚词堆砌可提升人类特征
- **结构对称性检测**：知网 3.0+ 新增对多章节结构镜像的识别。并行小节结构相似度 > 60% 会触发预警
- **Turnitin 中文检测预警**：Turnitin 宣布 2026 年 Q2 支持简体中文检测，届时中文学术论文将同时面临知网+Turnitin 双链路检测

**对抗要点**：
- 避免连续使用"基于…"、"本文研究…"等模板化表达
- 补充数据来源和具体实验细节（AI 常缺失）
- 打乱段落内部的"背景→方法→结论"线性结构
- 打破多章节结构镜像对称（详见策略三 3.5）
- 允许适当的"不完美"：思维跳跃、前后矛盾修正、非线性论述
- 每增加 100 字至少包含 1 处认知特征或 2 个具体细节

#### 万方检测

- **特点**：比对库较小，无跨语言检测能力，相对温和
- **薄弱点**：不涉及英文数据库，中英文混排内容检测较弱
- **对抗要点**：核心内容用规范中文表述，避免过度依赖外文引用

#### 维普检测

- **特点**：与知网类似但数据库覆盖有差异，同样新增 AIGC 检测服务
- **对抗要点**：与知网策略一致，注意段落逻辑的自然过渡；对句长均匀度特别敏感

### 5.2 PaperPass 查重与 AIGC 检测

#### 检测原理

PaperPass 采用**多模态比对引擎**：

**核心算法**：
1. **动态指纹越级扫描技术**：提取文本的多维特征指纹
2. **滑动窗口算法**：连续 7-13 个字符匹配即触发标记
3. **向量空间模型（VSM）+ 注意力机制**：多层次文本匹配
4. **深度学习语义识别**：检测段落结构相似性和"换词不换意"
5. **Transformer 架构**：捕捉上下文关联

**比对库**：自建 2000万+ 学术文献库，600亿+ 网页资源，每日更新 5000万+ 文献

**可视化报告（四色标注法）**：
- 红色：高重复（>=80%）| 橙色：中等重复（50-80%）
- 黄色：低重复（30-50%）| 绿色：安全区域

#### 2025 年更新

- 新增**降 AIGC 率**功能，支持查重与降 AIGC 同步优化
- 动态指纹技术升级，针对知网 3.0 新要求进行适配
- 支持中英混合论文检测

#### PaperPass 对抗要点

1. **核心策略**：打断连续字符匹配（插入词、拆分句子）
2. 同义词替换不够——需要**改变句法结构**
3. 主动被动转换、调整语序、改变论述角度
4. 添加个人分析：在引用后加入个人理解和评述
5. 初稿用 PaperPass 自查，定稿用学校指定系统验证

### 5.3 定向对抗策略速查表（中文平台）

| 检测器 | 核心技术 | 准确率(实测) | 最敏感维度 | 定向策略 |
|--------|---------|-------------|-----------|---------|
| **知网 AIGC 3.0** | 语义指纹+双链路检测 | 98.6% | 段落逻辑链条、套路化表达 | 消灭模板句式、补充数据来源、引入主观评价、打乱线性结构、允许"不完美" |
| **万方** | 比对库较小 | 中 | 段落结构模式 | 规范中文表述为主 |
| **维普** | 与知网类似 | 中高 | 句长均匀度、连接词频率 | 同知网策略，特别注意句长变化丰富 |
| **PaperPass** | 动态指纹+Transformer | - | 连续字符匹配（7-13字） | 打断连续字符、改变句法结构、添加个人分析 |

## Agent 执行步骤

### 第一步：文本诊断（含"六维 AI 指纹诊断法"）

1. 通读全文，提取**核心专业术语**，加入保护名单（改写时原样保留）
2. 扫描以下 AI 指纹并记录位置：
   - 套话开头（策略一 1.1 中的模式）
   - AI 高频过渡词（策略一 1.2）
   - AI 高频词汇（策略二 2.1-2.2 黑名单）
   - 连续 3 句以上句长相近的段落（低突发性）
   - 缺少第一人称/让步/反驳的段落
   - 模板化句式（如"基于…分析"、"本文旨在…"）
   - **结构对称性**：并行小节（如多个技术对比、多组实验）是否采用相同内部结构
3. 评估全文的查重风险段落（通用性描述、方法论套话）

#### 六维 AI 指纹诊断法

| 维度 | AI 特征 | 人类特征 | 快速检测方法 |
|------|--------|---------|------------|
| **困惑度** | 过于流畅、可预测 | 有跳跃、有意外用词 | 读一段，预判下一句——如果能准确猜中，说明困惑度低 |
| **突发性** | 句长均匀 | 长短交替、有节奏感 | 统计一段中各句字数，标准差 < 30% 即为低突发性 |
| **词汇密度** | 虚词多、实词少，密度偏低 | 实词密集、信息量大 | 数一段中名词/动词/形容词占比，低于 40% 即为偏低 |
| **连接词模式** | "首先/其次/最后"标准化 | 灵活、偶尔省略 | 搜索"首先""其次""综上所述"出现频次 |
| **个性化程度** | 无第一人称、无主观判断 | "笔者认为""据笔者观察" | 搜索"笔者""我认为"等个人标记词 |
| **具体性** | 空泛表述多 | 有具体数据、案例 | 搜索"显著影响""具有重要意义"等套话 |

### 第二步：分段改写（分轮次工作流）

按段落逐段处理，综合应用策略一至五：

**第一轮：结构层改写**（消灭最明显的 AI 痕迹）
1. 消灭 AI 套话和高频词（策略一、二）
2. 制造句式长短交替（策略一 1.3-1.4）
3. **打破结构对称性**：对并行小节执行错位排列、角度转换或详略交替（策略三 3.5）

**第二轮：语义层改写**（降低查重率+提升困惑度）
4. 对高查重风险段落执行句式重构（策略三 3.1-3.3）
5. 运用语言学技巧增加表达陌生感（策略三 3.4）
6. 处理特殊内容（表格、公式、代码描述文字、测试用例）（策略三 3.6）

**第三轮：个性化改写**（注入人类特征）
7. 在关键论述处注入学术个性标记（策略四）
8. **保留已有个人化痕迹**：括号吐槽、项目黑话、具体数字不得抹除（策略四 4.7）
9. 针对目标检测器执行定向对抗（策略五）

**第四轮：校验**（确保改写质量）
10. 确保每段改写后**原意零损失**，学术准确性不变
11. 检查术语保护名单中的词汇是否被改动
12. 检查是否误删了口语化表达或个人化痕迹

### 第三步：输出结果

1. **直接输出改写后的全文**
2. 在文末附上简短的 Markdown 修改说明列表，包含：
   - 替换了哪些 AI 高频词
   - 拆分/重组了哪些句子
   - 在哪些位置注入了人类写作标记
   - 运用了哪些语言学改写技巧
   - 对目标检测器的定向对抗措施
   - 对查重率可能产生影响的主要改动

## 注意事项

- **学术准确性优先**：任何改写都不得改变原文的学术含义和逻辑关系
- **术语零修改**：保护名单中的专业术语、人名、数据必须原样保留
- **不过度口语化**：保持学术论文应有的正式程度，只在衔接处适度自然化
- **不添加虚假信息**：不得凭空编造数据、引用或事实
- **不故意制造错误**：不使用错别字、语法错误等手段（这对学术论文不可接受）
- **伦理边界**：本技能旨在帮助学术作者将 AI 辅助写作的文本优化为符合个人学术风格的作品，而非鼓励学术不端。建议遵循"预查-优化-复检"流程，确保论文核心研究内容（研究设计、数据分析、核心结论）由作者独立完成。AI 仅用于辅助整理与语言润色。

## 常见十大误区

| 误区 | 正确做法 |
|------|---------|
| **只看总 AI 率，不看分布** | 逐章逐段看 AI 率标记，先处理 60% 以上的 |
| **用同义词替换法降 AI** | 改句子结构，不是改单个词 |
| **改得越多越好** | 精准处理，只改需要改的部分，30% 以下不用动 |
| **迷信"一键降 AI"** | 工具+人工配合，没有工具能完全替代人工 |
| **降 AI 后不再检测** | 改后必须再检测，有时工具处理反而引入新 AI 特征 |
| **把整篇丢给工具就完事** | 工具处理必须人工核查，特别是专业内容 |
| **用免费工具对付完事** | 免费工具处理字数极少或改写质量极低 |
| **降 AI = 降重** | 是两个不同目标，方法也不同 |
| **追求 0% AI 率** | 10% 以下足够，0% 反而可疑 |
| **不同平台结果可换算** | 知网 25% != 维普 25%，不可直接换算 |

## 无效方法清单（已被新型检测器识破）

| 无效方法 | 为什么失效 | 新型检测器如何识别 |
|---------|-----------|-----------------|
| **同义词替换** | 检测器看的是句式和语义模式，不是单个词 | 知网 2025+ 采用段落逻辑链条分析，词汇替换不影响判定 |
| **简单调整语序** | 不改变统计特征 | 突发性、困惑度指标无实质变化 |
| **用 QuillBot 改写** | Turnitin 2024+ 已专门识别 QuillBot 特征 | Turnitin 文档明确提及可检测 QuillBot 改写内容 |
| **让 AI 用"更人性化"的 prompt 生成** | 模型级的统计特征仍然存在 | 底层概率分布不变 |
| **仅替换连接词** | 连接词只是检测维度之一 | 句式结构、段落逻辑仍然是 AI 特征 |
| **全文一次性工具处理** | 可能引入新的 AI 特征 | 改写后的文本仍有统计规律性 |
| **翻译循环法**（中文->外文->中文） | 现代检测器能识别其语法痕迹 | 朱雀等检测器采用对抗性训练框架，对此免疫 |
| **Humanizer 工具改写** | Turnitin 2025 新增 AI Bypasser Detection，专门识别人性化工具处理痕迹 | 简单同义替换+轻度改写会被标记为"经 bypasser 修改的 AI 文本" |

## 学科差异化策略速查表

### 理工科

| 策略 | 具体方法 | 效果 |
|------|---------|------|
| **重点章节策略** | 优先处理绪论和文献综述，实验部分次之 | 投入产出比最高 |
| **代码和公式保护** | 提交前删除代码块和公式，处理后恢复 | 避免误改，省费用 |
| **实验细节注入** | 多加入自己的实验细节和数据分析过程 | AI 率降低 20-30% |
| **表格描述优化** | 检查表格上方的描述文字（常被忽略但 AI 率高） | 精准降 AI |

**理工科专属检查清单**：
- [ ] 代码块和数学公式是否已保护？
- [ ] 实验数据是否准确无误？
- [ ] 算法名称是否被误改？
- [ ] 图表描述是否补充了具体解读？
- [ ] 文献综述中"XX 提出"的句式是否已多样化？
- [ ] 技术定义是否加入了项目具体上下文（避免教科书式表述）？
- [ ] 多个技术/实验小节的结构是否镜像对称？是否已错位打破？
- [ ] 测试用例描述是否有个人处理经验和异常记录（避免格式完全统一）？

### 文科

| 策略 | 具体方法 | 效果 |
|------|---------|------|
| **学术史叙事化** | 将文献梳理转化为"问题提出->观点演进->学派形成->当代发展"的叙事 | AI 率降低 45-55 个百分点 |
